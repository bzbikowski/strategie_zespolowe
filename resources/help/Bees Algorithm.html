Schema:
  • begin
  • Generate randomly starting population of solutions
  • repreat
  • Evaluate the quality of solutions
  • Select places to search the neighborhood
  • Choose recruits for selected sites (more bees for the best sites) and evaluate them
  • Select the best bee in each area
  • Assign random movements to the remaining bees and evaluate them
  • until end criterion is not met
  • end
Parameters:
  • number_of_scouts -> number of scout bees(n)
  • number_of_chosen_places -> number of places selected n of those visited(m)
  • number_of_best_places -> number of best areas for the m places visited(e)
  • number_of_recruits_for_best -> number of recruiters for the best e areas (nep)
  • number_of_recruits_for_other -> number of recruiters for other (m-e) selected areas (nsp)
  • init_size_of_area -> the initial size of the area for random movement (ngh)
  • init_size_of_neighbourhood -> the initial size of the area for neighbourhood search (nnh)
  • bee_search_prob -> the probalility that the selected will perform random movement
